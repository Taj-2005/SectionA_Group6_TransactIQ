# WildChat Data Dictionary

This document details the dataset structures used in the WildChat Analytics project, distinguishing between the raw dataset obtained from Hugging Face and the processed datasets generated by our pipeline. This ensures a consistent understanding of KPIs across the Executive Summary, Operational Intelligence, and Safety Trust dashboard sections.

## 1. Raw Dataset Metadata

- **Source**: WildChat-1M (AllenAI, Hugging Face) — `https://huggingface.co/datasets/allenai/WildChat-1M`
- **Dataset variants**: WildChat-1M
- **Raw format**: Streaming extraction from HuggingFace Hub, saved as raw parquet/JSON.
- **Grain (row-level meaning)**: One row = one conversation containing a list of message turns.
- **Primary keys**: `conversation_id`

## 2. Processed Datasets

These datasets are located in `data/processed/` and are directly used for our EDA, feature analysis, and the dashboard.

### 2.1. `conversations_clean.csv`
**Grain**: Conversation-level summary.
| Column | Type | Example | Business meaning |
|---|---|---|---|
| `conversation_id` | string | `hash_a1b2...` | Unique identifier for the conversation |
| `turn_count` | int | `5` | Number of turns (user+assistant messages) |
| `start_time` | datetime | `2024-01-01 10:00:00` | Start of conversation |
| `end_time` | datetime | `2024-01-01 10:05:00` | End of conversation |
| `duration_seconds` | float | `300.5` | Total duration of the interaction |
| `user_message_count`| int | `2` | Number of messages sent by the user |
| `assistant_message_count` | int | `3` | Number of messages sent by the AI |
| `avg_sentiment_score`| float | `0.45` | Average VADER sentiment across user messages |
| `toxicity_score` | float | `0.1` | Composite toxicity score (OpenAI moderation) |
| `country` | category | `US` | Geographic country of origin |
| `state` | category | `CA` | Geographic state of origin |
| `model` | category | `gpt-4` | AI model used |

### 2.2. `messages_sample.csv`
**Grain**: Individual message-level.
| Column | Type | Example | Business meaning |
|---|---|---|---|
| `conversation_id` | string | `hash_a1b2...` | Ties back to parent conversation |
| `turn_id` | int | `1` | Sequential identifier for the message |
| `role` | category | `user` / `assistant`| Who sent the message |
| `content` | string | `"Write a python script"`| Original message text |
| `content_clean` | string | `"write python script"`| Preprocessed text content |
| `token_count` | int | `14` | Number of tokens in message |
| `sentiment_score` | float | `0.2` | VADER sentiment score (user only) |
| `language` | category | `en` | Detected language code |
| `timestamp` | datetime | `2024-01-01 10:00:05`| Message timestamp |
| `toxic` | boolean | `False` | Boolean toxicity flag |

### 2.3. `user_segments.csv`
**Grain**: User-level behavioral segmentation.
| Column | Type | Business meaning |
|---|---|---|
| `segment` | category | `Power Users`, `Casual Users`, `Regular Users` based on activity |
| `conversation_frequency`| float | Rate of conversations over time |
| `avg_message_length`| float | Average tokens/characters per message |

### 2.4. `prompt_categories.csv`
**Grain**: Categorized User Prompts.
| Column | Type | Business meaning |
|---|---|---|
| `category` | category | E.g., `Coding and Technical`, `Harmful`, `Roleplay` |
| `confidence_score`| float | Clustering confidence score |

### 2.5. Additional Aggregations
- **`geo_summary.csv`**: Country and state-level aggregations (counts, avg sentiment, toxicity rates).
- **`daily_kpis.csv`**: Time-series aggregation (daily volume, sentiment trends, safety incidents).

## 3. Dashboard KPI Mapping

To ensure consistency, KPIs across our three primary dashboard sections must be computed using the fields defined above.

### A. Executive Summary
- **Total Conversations** = count(`conversation_id`) from `conversations_clean.csv`
- **Total Messages** = sum(`user_message_count` + `assistant_message_count`)
- **Top Models Used** = grouped counts on `model` in `conversations_clean.csv`
- **Daily Active Volume** = metric directly from `daily_kpis.csv`

### B. Operational Intelligence
- **Average Conversation Duration** = mean(`duration_seconds`)
- **Average Turns per Conversation** = mean(`turn_count`)
- **User Segment Distribution** = counts grouped by `segment` in `user_segments.csv`
- **Prompt Category Distribution** = counts grouped by `category` in `prompt_categories.csv`
- **Average Tokens per Message** = mean(`token_count`) from `messages_sample.csv`

### C. Safety Trust
- **Overall Toxicity Rate** = percentage of rows where `toxic == True` in `messages_sample.csv` or average of `toxicity_score` in `conversations_clean.csv`
- **Toxicity by Model** = mean(`toxicity_score`) grouped by `model`
- **Toxicity by Prompt Category** = cross-reference `prompt_categories.csv` and `toxic` flag
- **Average User Sentiment** = mean(`avg_sentiment_score`)
