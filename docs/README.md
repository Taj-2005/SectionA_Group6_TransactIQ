# `docs/`

Project documentation (source of truth).

- `_source/`: original capstone brief and converted text
- `architecture/`: architecture diagrams and system overview
- `adr/`: Architecture Decision Records (ADRs)
- `dictionaries/`: data dictionary and KPI dictionary
- `stakeholders/`: stakeholder map, requirements, KPI ownership
- `glossary/`: shared terminology (metrics, stages, model terms)
- `project/`: project charter, scope, assumptions
